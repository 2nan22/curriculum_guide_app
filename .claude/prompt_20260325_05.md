# Session 5: Backend Logging

## 목표
백엔드 전체에 구조화된 로깅 시스템을 도입합니다.
자정 로테이션, 최대 100개 보관, 콘솔+파일 동시 출력을 구현합니다.

## 현재 아키텍처
```
backend/app/
├── main.py                  # FastAPI 앱 진입점, CORS, 헬스체크
├── config.py                # pydantic-settings (log_level: str = "INFO" 이미 있음)
├── models/schemas.py
├── routers/
│   ├── roadmap.py           # POST /api/roadmap/generate[/stream]
│   ├── node.py              # POST /api/node/detail, /api/node/quiz  (인메모리 캐시)
│   └── chat.py              # POST /api/chat/stream (SSE)
├── services/
│   ├── llm_service.py       # OllamaProvider / UpstageProvider
│   └── rag_service.py       # load_guideline() lru_cache
└── utils/
    ├── parser.py            # extract_json, validate_roadmap, fallback_roadmap
    └── prompts.py
```

## Task 1: 로거 모듈 생성

### `backend/app/utils/logger.py` 신규

```python
# 구현 요구사항
- get_logger(name: str) -> logging.Logger  # 모듈별 logger 반환 팩토리
- setup_logging()  # 앱 시작 시 1회 호출, 이후 get_logger()만 사용
```

**핸들러 구성:**
- `StreamHandler` — 콘솔 출력 (개발 편의)
- `TimedRotatingFileHandler`
  - `when="midnight"`, `interval=1`
  - `backupCount=100` — 최대 100개 보관 (101번째부터 오래된 파일 자동 삭제)
  - 파일 경로: `logs/aipath.log`
  - 로테이션된 파일: `logs/aipath.log.YYYY-MM-DD`
  - `encoding="utf-8"`

**로그 포맷:**
```
2026-03-25 14:32:01,123 | INFO  | roadmap     | [POST /generate] role=Backend level=Mid
```
형식: `%(asctime)s | %(levelname)-5s | %(name)-12s | %(message)s`

**로그 레벨:** `settings.log_level` 값 사용 (`INFO` 기본)

**`logs/` 디렉터리:** 존재하지 않으면 자동 생성 (`mkdir(parents=True, exist_ok=True)`)

**`.gitignore`에 `logs/` 추가** (로그 파일 커밋 방지)

## Task 2: main.py — 앱 시작 및 요청/응답 미들웨어 로깅

```python
# setup_logging() 앱 시작 시 1회 호출
# 로그 위치: lifespan 이벤트 또는 모듈 최상단

# 로그 항목
INFO  앱 시작: "AI Path API starting — provider={provider} model={model} env={app_env}"
INFO  요청: "{method} {path}" (미들웨어 또는 각 엔드포인트 시작)
INFO  응답: "{method} {path} → {status_code} ({elapsed_ms}ms)"
ERROR 처리되지 않은 예외: exc_info=True
```

FastAPI `@app.middleware("http")`로 모든 요청/응답 자동 로깅.
