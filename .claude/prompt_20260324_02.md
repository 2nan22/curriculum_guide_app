## Task 1: LLM Abstraction Layer

LLM 서비스 추상화 레이어를 구현해주세요.

### 요구사항
- `src/services/llmService.js` 구현
  - Provider 전환 지원: `ollama` (로컬) / `upstage` (클라우드)
  - 환경변수 `.env`로 관리:
    ```
    VITE_LLM_PROVIDER=ollama        # 또는 upstage
    VITE_OLLAMA_BASE_URL=http://localhost:11434
    VITE_UPSTAGE_API_KEY=your_key
    VITE_UPSTAGE_MODEL=solar-pro
    ```
  - `generateRoadmap(role, level, guidelineData)` 함수 export
  - `streamChat(messages, onChunk)` 함수 export (AI Tutor용 스트리밍)
- Ollama CORS 문제 해결:
  - `public/ollama-setup.md` 안내 문서 작성 (`OLLAMA_ORIGINS=*` 환경변수 설정 방법)
  - 연결 실패 시 사용자에게 친절한 에러 메시지 표시
- `src/services/upstageService.js` - Upstage Solar API 호출 래퍼
- `src/hooks/useLLM.js` - 로딩/에러/데이터 상태 관리 커스텀 훅

### 커밋 메시지
```

- Ollama CORS 해결 안내 및 연결 실패 에러 처리
- useLLM 커스텀 훅으로 로딩/에러 상태 관리
```

## Task 2: Dynamic Roadmap Generation

LLM을 활용한 동적 로드맵 생성 로직을 구현해주세요.

### 요구사항
- `src/utils/prompts.js` - System Prompt 설계
  - 역할: "당신은 IT 커리큘럼 전문가입니다"
  - 직무/레벨 조합별 최적화된 프롬프트 템플릿
  - 반드시 아래 JSON 스키마로만 응답하도록 강제:
    ```json
    {
      "root": {
        "id": "root",
        "label": "Backend Engineer",
        "children": [
          {
            "id": "branch_1",
            "label": "핵심 언어",
            "children": [
              { "id": "leaf_1_1", "label": "Java / Spring Boot", "description": "...", "estimatedWeeks": 4 },
              { "id": "leaf_1_2", "label": "Python / FastAPI", "description": "...", "estimatedWeeks": 3 }
            ]
          }
        ]
      }
    }
    ```
- `src/utils/roadmapParser.js` - LLM 응답 파싱
  - JSON 추출 (마크다운 코드블록 제거)
  - 스키마 유효성 검증
  - 파싱 실패 시 fallback 데이터 반환
- `RoadmapPage.jsx` - 생성 흐름 구현
  - 로딩 상태: 스켈레톤 UI 또는 진행 애니메이션
  - 성공 시 MindmapLayout에 데이터 바인딩
  - 에러 시 재시도 버튼 표시

### 커밋 메시지
```
feat: LLM 기반 동적 로드맵 생성 및 데이터 바인딩

- 직무/레벨별 System Prompt 템플릿 설계
- LLM 응답 JSON 파싱 및 스키마 검증 로직 구현
- RoadmapPage 로딩/성공/에러 상태 UI 처리
```

## Task 3: Knowledge Base (RAG Lite)

표준 커리큘럼 데이터를 LLM 컨텍스트에 주입하는 로직을 구현해주세요.

### 요구사항
- `public/data/curriculum_guideline.json` 작성
  - 각 직무별 필수 학습 도메인 및 키워드 정의
  - 예시 구조:
    ```json
    {
      "Backend": {
        "Junior": ["HTTP/REST API", "DB CRUD", "Git", "Spring Boot 기초"],
        "Mid": ["JPA/ORM", "캐싱(Redis)", "메시지큐", "TDD", "Docker"],
        "Senior": ["대규모 시스템 설계", "MSA", "성능 최적화", "팀 리딩"]
      },
      "Frontend": { ... },
      "AI/ML": { ... }
    }
    ```
- `src/utils/ragUtils.js` - 가이드라인 데이터 로드 및 프롬프트 삽입
  - `fetchGuideline(role, level)` - 해당 직무/레벨 가이드라인 반환
  - `buildContextualPrompt(role, level, guideline)` - 가이드라인이 포함된 최종 프롬프트 생성
- 생성된 로드맵 노드가 가이드라인 키워드를 포함하는지 검증하는 로직 추가

### 커밋 메시지
```
feat: 커리큘럼 가이드라인 JSON 및 RAG Lite 컨텍스트 주입 구현

- curriculum_guideline.json에 직무/레벨별 필수 학습 키워드 정의
- ragUtils.js로 가이드라인 데이터를 LLM 프롬프트에 주입
- 생성 로드맵의 가이드라인 키워드 포함 여부 검증
```
