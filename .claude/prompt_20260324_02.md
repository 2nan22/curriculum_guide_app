## 목표
백엔드 LLM 엔진을 완성하고 프론트엔드에서 실시간 로드맵 데이터를 생성합니다.

## 현재 아키텍처 (Session 1 완료 기준)
- LLM 호출: **백엔드** `backend/app/services/llm_service.py` (Ollama/Upstage 추상화 완성)

- 더미 데이터: `SelectionPage.jsx`의 `buildDummyRoadmap()` → 이번 세션에서 실제 API 호출로 교체

## Task 1: 백엔드 LLM 프롬프트 & 파싱 완성

`backend/app/routers/roadmap.py`의 프롬프트와 파싱 로직을 강화해주세요.

### 요구사항
- `backend/app/utils/prompts.py` 신규 파일로 프롬프트 분리
  - `build_system_prompt(level: str) -> str`
  - `build_user_prompt(role: str, level: str, guideline: dict) -> str`
  - 레벨별 톤앤매너 지시 포함 (Junior: 친근, Mid: 실무, Senior: 전문)
- `backend/app/utils/parser.py` 신규 파일로 파싱 로직 분리
  - `extract_json(raw: str) -> dict` — 마크다운 코드블록 제거 후 JSON 파싱
  - `validate_roadmap(data: dict) -> bool` — 필수 필드(`root.children`) 검증
  - 파싱 실패 시 `fallback_roadmap(role, level)` 반환
- `roadmap.py` 라우터에서 위 유틸 함수 사용하도록 리팩터링

### 커밋 메시지
```
feat: 백엔드 LLM 프롬프트 분리 및 JSON 파싱/검증 강화

- prompts.py: 레벨별 톤앤매너 포함 프롬프트 빌더 분리
- parser.py: LLM 응답 JSON 추출, 스키마 검증, fallback 로직 구현
- roadmap.py 라우터에서 유틸 함수 적용
```

## Task 3: RAG Lite — 커리큘럼 가이드라인 주입 강화

`public/data/curriculum_guideline.json` 데이터가 실제로 로드맵 품질에 반영되는지 검증하고 강화합니다.

### 요구사항
- `backend/app/services/rag_service.py` 신규 파일
  - `load_guideline(data_dir: str) -> dict` — JSON 파일 로드 (캐싱)
  - `get_keywords(guideline: dict, role: str, level: str) -> list[str]` — 해당 직무/레벨 키워드 추출
  - `inject_context(prompt: str, keywords: list[str]) -> str` — 키워드를 프롬프트에 구조적으로 주입
- `roadmap.py` 라우터에서 `rag_service` 활용
- `public/data/curriculum_guideline.json` 키워드 보강 (직무별 30개 이상)

### 커밋 메시지
```
feat: RAG Lite 서비스 분리 및 커리큘럼 가이드라인 키워드 보강

- rag_service.py로 가이드라인 로드/캐싱/주입 로직 분리
- curriculum_guideline.json 직무별 키워드 30개 이상으로 확장
- 라우터에서 rag_service 연동 완료
```
